# Lux Log 写作与维护指南

这份指南教你**怎么写文章、插图片、加标签、出合集、发布上线**。
日常更新基本只会用到 `posts/` 目录和 `config.js` 两个地方。

> 小词典：本指南里说的「站点根目录」指 `D:\luxuuuuuuuu.github.io\`，
> 也就是有 `index.html` 的那一层。

## 一、整体结构：谁管什么

| 你想改的东西 | 去哪改 |
|------------|-------|
| 写文章、改文章 | `posts/` 里的 `.md` 文件 |
| 哪些文章会显示 | `posts/posts.json` |
| 站点标题、背景图、个人简介、联系方式、评论、统计 | `config.js` |
| 图片（背景图、封面、插图） | `assets/` 目录 |
| 配色、字体、布局 | `styles.css`（一般不用动） |

## 二、添加一篇新文章

### 方式 A：用脚本（推荐）

在站点根目录打开终端（Git Bash），运行：

```bash
./new-post.sh "我的文章标题"
```

它会自动：
- 在 `posts/` 里生成一个 `.md` 文件（带好头部模板）
- 把这个文件登记进 `posts/posts.json`

然后用编辑器（如 VS Code）打开那个新 `.md`，把正文写进去即可。

> 想自定义文件名（网址里的 slug），可以加第二个参数：
> `./new-post.sh "我的标题" my-first-post` → 生成 `posts/my-first-post.md`

### 方式 B：手动（不想用脚本时）

1. 在 `posts/` 里新建一个文件，例如 `posts/my-post.md`
2. 在文件开头写好[头部信息](#三文章头部信息frontmatter)，下面写正文
3. 打开 `posts/posts.json`，把新文件名加进去（**放在最前面 = 显示在最上面**）：

```json
[
  "my-post.md",
  "discrete-math-thinking.md"
]
```

> ⚠️ `posts.json` 是 JSON 格式：每个文件名用英文双引号，名字之间用英文逗号隔开，
> **最后一个名字后面不要加逗号**，否则页面会读不出文章。

## 三、文章头部信息（frontmatter）

每篇 `.md` 文件**最上面**用两行 `---` 包起来的部分，是这篇文章的信息：

```markdown
---
title: 离散数学 思维与挑战
date: 2026-06-18
tags: 读书, 离散数学
excerpt: 这里写一句话摘要，会显示在文章卡片上。
cover: assets/discrete-cover.jpg
---

正文从这里开始……
```

| 字段 | 作用 | 说明 |
|------|------|------|
| `title` | 文章标题 | 必填 |
| `date` | 日期 | 格式 `年-月-日`，例如 `2026-06-18`。**文章按日期从新到旧排序** |
| `tags` | 标签 | 用英文逗号分隔，例如 `读书, 离散数学`。留空就没有标签 |
| `excerpt` | 摘要 | 显示在文章卡片和「精选文章」上，建议一句话 |
| `cover` | 封面图 | 可留空（`cover:` 后面什么都不写）。填了就会显示封面 |

> 阅读时长不用自己填，系统会根据正文长度自动算。

## 四、Markdown 写法速查

正文用 Markdown 写。下面左边是你写的，右边是显示效果：

```markdown
# 一级标题
## 二级标题
### 三级标题

普通段落，**加粗**，*斜体*，`行内代码`。

- 无序列表项
- 第二项

1. 有序列表
2. 第二步

> 这是一段引用

[这是一个链接](https://github.com/LuXuuuuuuuu)

---（上面三个连字符是一条分隔线）

​```
这里是代码块
可以放多行代码
​```
```

要点：
- **段落之间空一行**，否则会被当成同一段连起来。
- 标题的 `#` 后面要有一个空格：`## 标题`（不是 `##标题`）。
- emoji 直接写就行 👍😂，会正常显示。

## 五、插入图片（封面 + 正文插图）

### 第 1 步：把图片放进 `assets/`

把图片文件复制到站点根目录的 `assets/` 文件夹里，例如 `assets/cat.jpg`。
支持 jpg / png / webp / gif / svg。

### 第 2 步：在文章里引用

**封面图**（显示在文章顶部和卡片上）——写在头部的 `cover`：

```markdown
---
title: 我的文章
cover: assets/cat.jpg
---
```

**正文插图**——在正文里需要的位置写：

```markdown
![一只猫](assets/cat.jpg)
```

其中方括号 `[]` 里是图片说明（图片加载不出来时会显示这段文字）。

> ⚠️ **路径一律用 `assets/xxx.jpg`，不要写成 `../assets/`，也不要用反斜杠 `\`。**
> 因为文章是被嵌进首页显示的，路径是相对「站点根目录」算的。

## 六、标签怎么加

在文章头部的 `tags` 里写，用英文逗号分隔：

```markdown
tags: 读书, 离散数学, 随笔
```

效果：
- 文章卡片和详情页会显示这些标签小圆角块。
- 博客区顶部会自动出现标签筛选按钮，点一下只看该标签的文章。
- 标签也能被搜索到。

> 建议标签别太碎，几个固定的大类更好用（例如 `读书` / `cs` / `生活` / `离散数学`）。

## 七、出一个「合集 / 系列」

本站的「合集」就是**用同一个标签**把一组文章串起来——不需要额外功能。

做法：
1. 给同一系列的每篇文章，都加上一个**相同的标签**，例如 `tags: 离散数学`。
2. 读者点博客区顶部那个 `离散数学` 标签按钮（或在搜索框输入「离散数学」），
   就能看到这个合集下的所有文章。

排序：合集里的文章按 `date` 从新到旧排。如果你想让系列**按阅读顺序（第 1 篇在前）**，
有两个简单办法：
- 给标题加序号，例如 `（一）集合论`、`（二）群论`；
- 用日期控制顺序：想排在前面的，把 `date` 设得更晚一点。

> 如果以后你想要更正式的「系列页 + 上一篇/下一篇」导航，告诉我，我可以加上。

**标题 / 摘要 / 标签 / 正文**里查找并过滤。无需配置，已经能用。

## 九、发布上线

改完文章或配置后，在站点根目录运行：

```bash
./publish.sh "这次改了什么"
```

它会自动提交并推送到 GitHub。**约 1 分钟后**线上 https://luxuuuuuuuu.github.io 自动更新。

> 引号里随便写这次的改动说明，例如 `./publish.sh "新增一篇读书笔记"`。

## 十、本地预览

想在发布前先看看效果。**注意不能直接双击 `index.html`**（浏览器安全限制会让文章读不出来），
要起一个本地小服务器：

```bash
# 在站点根目录运行（二选一）
python -m http.server 8000
# 或
npx serve .
```

然后浏览器打开 http://localhost:8000 。改完文件刷新页面即可看到变化。

## 十一、常见问题排错

**1. 改了文章，线上/本地没变化？**
浏览器缓存了旧页面。按 `Ctrl + Shift + R` 强制刷新，或用无痕窗口打开。

**2. 整个页面白屏 / 文章全不显示？**
多半是 `config.js` 或 `posts.json` 有语法错误（漏了逗号、引号、key）。
在根目录运行这条命令自检：

```bash
node -e 'global.window={};require("./config.js");console.log("config ok")'
```

能打印 `config ok` 就说明 `config.js` 没问题。`posts.json` 则检查逗号和引号是否成对。

**3. 图片显示不出来？**
- 确认图片确实在 `assets/` 里，文件名大小写一致（线上区分大小写）。
- 路径要写 `assets/xxx.jpg`，不要 `../assets/`，不要反斜杠 `\`。

**4. 背景图不显示？**
`config.js` 里 `background.image` 路径要用正斜杠 `/`，例如 `assets/page.jpg`。

**5. 访客统计数字不显示？**
不蒜子是免费公共服务，偶尔会抽风，属正常现象，过段时间会恢复。

**6. 评论框不出现？**
确认仓库已开启 Discussions、giscus 应用已安装，且 `config.js` 的
`comments` 里 `enabled: true` 且 `repoId` / `categoryId` 都填了。
